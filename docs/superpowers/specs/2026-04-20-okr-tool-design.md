## 2. 역할 구조

| 역할 | 설명 |
|------|------|
| **관리자 (Admin)** | 시스템 전체 관리자. 멤버 등록·권한 설정, 조직도 업로드, 사이클 생성, 전사 현황 조회 |
| **리뷰어 (Reviewer)** | 담당자의 목표 진행률 확인·코멘트. 부서장은 부서원의 리뷰어, 대표는 부서장의 리뷰어 |
| **담당자 (Member)** | 목표 수립·체크인 실행. 부서원 뿐 아니라 부서장도 대표 기준에서는 담당자 |

> 리뷰어-담당자 관계는 상대적 계층 구조. 각 유저에게 "나의 리뷰어"를 지정하는 방식으로 구현.

## 3. MVP 기능 범위

### 포함 (1차)
- 목표 목록 조회 및 등록
- 목표 상세 (측정 지표, 목표치, 사이클, 리뷰어 자동 지정)
- 체크인 (담당자: 진행률 입력, 상태 변경, 메모)
- 리뷰어 코멘트 (체크인에 대한 리뷰어 코멘트)
- 리뷰어 대시보드 (담당자별 진행률, 난항 강조)
- 관리자 화면 (멤버 권한 설정, CSV 조직도 업로드, 사이클 관리)
- Slack 알림 (내용 포함 알림)
- `/OKR` Claude Code 플러그인

### 제외 (2차)
- 피드백 — 360도 평가 (동료 간 상호 평가, 다면 평가)
- Slack에서 직접 체크인·코멘트 입력 (명령어 방식)

## 4. 인프라 & 기술 스택

| 항목 | 결정 |
|------|------|
| 웹앱 프레임워크 | Next.js 16 + TypeScript (기존 레포 유지) |
| 스타일 | Tailwind CSS 4 |
| ORM | Prisma |
| DB | PostgreSQL (사내 로컬 서버, 개발실 구축) |
| 인증 | NextAuth.js (역할: ADMIN / REVIEWER / MEMBER) |
| 배포 | 사내 로컬 서버 self-hosted (`npm run build && start`) |
| 외부 연동 | Slack (알림 전용), Flex HR (CSV 내보내기 → 업로드) |

### DB 역할 리네이밍
기존 Prisma 스키마의 `MANAGER` → `REVIEWER` 로 재정의.  
User 모델에 `reviewerId` 필드 추가하여 리뷰어-담당자 계층 관계 표현.

## 5. 화면 구성 (5개)

### 5-1. 목표 목록 (`/goals`)
- 내 목표 전체 리스트 (담당자 기준)
- 상태 필터: 순항 / 난항 / 대기 / 완료
- 진행률 % 표시
- 새 목표 등록 버튼
- 리뷰어는 담당자 선택 후 해당 목표 조회 가능

### 5-2. 목표 상세 / 등록 (`/goals/[id]`)
- 목표명, 설명
- 측정 지표 (숫자형 or 달성여부)
- 목표치 설정
- 사이클(분기) 선택
- 리뷰어: 시스템에서 자동 지정 (User.reviewerId 기준)
- 체크인 히스토리 목록

### 5-3. 체크인 (`/goals/[id]/checkin`)
- 현재 진행률 입력
- 상태 변경 (순항 / 난항)
- 이번 기간 활동 메모
- 제출 시 → 리뷰어에게 Slack DM 알림 발송

### 5-4. 리뷰어 대시보드 (`/dashboard`)
- 내가 리뷰하는 담당자 목록
- 담당자별 목표 진행률 카드
- 난항 목표 상단 강조 표시
- 각 체크인에 코멘트 입력 → 담당자에게 Slack DM 알림

### 5-5. 관리자 화면 (`/admin`)
- 멤버 목록 및 역할 설정 (ADMIN / REVIEWER / MEMBER)
- 리뷰어-담당자 매핑
- CSV 조직도 업로드 (Flex HR 내보내기 파일 기준, 필요 컬럼: 이름, 이메일, 부서, 직책, 리뷰어이메일)
- 사이클(분기) 생성·관리
- 전사 OKR 현황 종합 조회

## 7. `/OKR` Claude Code 플러그인

### 연결 방식
DB 직접 접근 (Prisma). 사내 내부망 전용이므로 별도 API 인증 불필요.  
DB 접속 정보는 `.env` 파일로 분리 (GitHub에 미포함).

### 제공 명령어
| 명령어 | 기능 |
|--------|------|
| `/OKR 목표 보기` | 내 목표 목록 조회 |
| `/OKR 목표 추가` | 새 목표 등록 |
| `/OKR 체크인` | 진행률 업데이트 |
| `/OKR 팀보기` | 팀 전체 현황 조회 (리뷰어 전용) |

### 배포
GitHub 공개 레포로 배포. 설치 후 `.env`에 DB 접속 정보 입력하여 사용.

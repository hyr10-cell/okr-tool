# OKR 도구 UAT 시나리오 및 테스트 케이스

## 1. 인증 (Authentication)

### 시나리오 1.1: 정상 로그인
**전제 조건:** 유효한 계정 정보 필요
1. 로그인 페이지(`/login`)에 접속
2. 이메일 입력: `admin@example.com`
3. 비밀번호 입력: `password`
4. "로그인" 버튼 클릭
5. **기대 결과:** 대시보드로 리디렉트, 사용자 정보 표시

### 시나리오 1.2: 잘못된 계정으로 로그인
1. 로그인 페이지에 접속
2. 유효하지 않은 이메일/비밀번호 입력
3. "로그인" 버튼 클릭
4. **기대 결과:** 에러 메시지 표시, 페이지 유지

### 시나리오 1.3: 빈 필드로 로그인 시도
1. 로그인 페이지에서 필드를 비워둔 상태
2. "로그인" 버튼 클릭
3. **기대 결과:** 필수 필드 경고 메시지, 로그인 거부

## 2. 목표 관리 (Goals)

### 시나리오 2.1: 새 목표 생성 ✅
**경로:** 대시보드 → 목표 → "+ 새 목표"
1. 목표 페이지 접속
2. "+ 새 목표" 버튼 클릭
3. 모달에서 다음 정보 입력:
   - 목표명: "API 보안 강화"
   - 설명: "프로덕션 환경의 보안 취약점 해결"
   - 시작일: 2024-04-01
   - 종료일: 2024-06-30
4. "만들기" 버튼 클릭
5. **기대 결과:**
   - 모달 닫힘
   - 목표 목록에 새 항목 추가됨
   - 목표 상태: 대기 (PENDING)

### 시나리오 2.2: 목표 편집 ✅
**전제 조건:** 목표가 존재해야 함
1. 목표 목록에서 목표 카드 클릭
2. 우측 사이드바에서 "수정하기" 버튼 클릭
3. 모달에서 다음 수정:
   - 목표명 변경: "API 보안 강화 → API 보안 강화 (우선순위 1)"
4. "수정하기" 버튼 클릭
5. **기대 결과:**
   - 모달 닫힘
   - 목표 목록과 사이드바에서 변경된 제목 확인

### 시나리오 2.3: 목표 필터링 ✅
**경로:** 목표 페이지
1. 목표 목록 위의 필터 영역 확인
2. 상태 필터 드롭다운에서 "순항" 선택
3. **기대 결과:** 상태가 "순항"인 목표만 표시
4. 레벨 필터 드롭다운에서 "회사" 선택
5. **기대 결과:** 상태 필터 + 레벨 필터 적용된 목표 표시
6. "필터 초기화" 버튼 클릭
7. **기대 결과:** 모든 필터 해제, 전체 목표 다시 표시

### 시나리오 2.4: 목표 검색
**경로:** 목표 페이지
1. 검색 필드에 "API" 입력
2. **기대 결과:** "API"를 포함하는 목표만 표시
3. 검색 필드 비우기
4. **기대 결과:** 모든 목표 다시 표시

### 시나리오 2.5: 필수 필드 검증
**경로:** 목표 페이지 → 새 목표 생성
1. "+ 새 목표" 클릭하여 모달 열기
2. 필드를 비우고 "만들기" 클릭
3. **기대 결과:** "목표명을 입력해주세요" 경고
4. 목표명만 입력하고 "만들기" 클릭
5. **기대 결과:** "기간을 입력해주세요" 경고

## 3. 체크인 (Check-in)

### 시나리오 3.1: 체크인 제출 ✅
**전제 조건:** 목표 존재
1. 목표 목록에서 목표 클릭
2. 우측 사이드바에서 "체크인하기" 버튼 클릭
3. 체크인 모달에서:
   - 상태: "순항" 선택
   - 코멘트: "현재 진행 상황 설명..." 입력
4. "제출하기" 버튼 클릭
5. **기대 결과:**
   - 모달 닫힘
   - (향후 체크인 이력에 추가될 예정)

### 시나리오 3.2: 정량지표 포함 체크인
**전제 조건:** 목표에 정량지표가 설정되어 있어야 함
1. 체크인 모달에서:
   - 상태: "순항" 선택
   - 현재값: "85" 입력 (정량지표 필드)
   - 코멘트 입력
2. "제출하기" 클릭
3. **기대 결과:** 정량지표 값이 저장됨

### 시나리오 3.3: 상태 변경을 통한 체크인
**경로:** 체크인 모달
1. 상태 드롭다운 확인:
   - 대기
   - 순항
   - 난항
   - 완료
   - 중단
2. 각 상태 선택 가능 확인
3. **기대 결과:** 모든 상태 옵션 표시

## 4. 가중치 관리 (Weight Management)

### 시나리오 4.1: 목표 가중치 설정 ✅
**경로:** 목표 페이지 → "⚖️ 가중치 설정"
1. "⚖️ 가중치 설정" 버튼 클릭
2. 모달에서 각 목표의 가중치 입력:
   - 목표1: 30%
   - 목표2: 40%
   - 목표3: 30%
3. 실시간 업데이트 확인:
   - 총 가중치: 100% (녹색 표시)
   - 남은 가중치: 0%
4. "저장하기" 버튼 클릭
5. **기대 결과:** 모달 닫힘, 가중치 저장

### 시나리오 4.2: 가중치 오버 경고
**경로:** 가중치 설정 모달
1. 목표 가중치들의 합이 100%를 초과하게 입력:
   - 목표1: 60%
   - 목표2: 50%
2. **기대 결과:**
   - 총 가중치: 110% (빨간색)
   - 경고 메시지: "총 가중치가 100%를 초과했습니다"
   - "저장하기" 버튼 비활성화

### 시나리오 4.3: 가중치 미할당 경고
**경로:** 가중치 설정 모달
1. 목표 가중치들의 합이 100%보다 작게 입력:
   - 목표1: 30%
   - 목표2: 40%
2. **기대 결과:**
   - 총 가중치: 70% (주황색)
   - 정보 메시지: "아직 30%의 가중치를 할당하지 않았습니다"
   - "저장하기" 버튼 활성화 (경고만, 제출은 가능)

## 5. 피드백 (Feedback)

### 시나리오 5.1: 피드백 작성 ✅
**경로:** 피드백 페이지 → "+ 피드백 작성"
1. "+ 피드백 작성" 버튼 클릭
2. 모달에서:
   - 받는 사람: "김팀장" 입력
   - 유형: "좋은 점 👍" 선택
   - 내용: "좋은 진행 방향입니다. 더 빠른 속도로 진행해주세요." 입력
3. "작성하기" 버튼 클릭
4. **기대 결과:**
   - 모달 닫힘
   - 피드백이 목록에 추가됨
   - 방향: "보낸 피드백"으로 표시

### 시나리오 5.2: 피드백 필터링 - 방향별
**경로:** 피드백 페이지
1. 방향 필터에서 "받은 피드백" 선택
2. **기대 결과:** direction이 "received"인 피드백만 표시
3. "보낸 피드백" 선택
4. **기대 결과:** direction이 "sent"인 피드백만 표시
5. "전체" 선택
6. **기대 결과:** 모든 피드백 표시

### 시나리오 5.3: 피드백 필터링 - 유형별
**경로:** 피드백 페이지
1. 유형 필터에서 "좋은 점" 선택
2. **기대 결과:** type이 "KEEP_GOING"인 피드백만 표시
3. "개선점" 선택
4. **기대 결과:** type이 "IMPROVE"인 피드백만 표시

### 시나리오 5.4: 복합 필터 적용
**경로:** 피드박 페이지
1. 방향: "받은 피드백"
2. 유형: "좋은 점"
3. **기대 결과:** 두 조건을 모두 만족하는 피드백만 표시

### 시나리오 5.5: 필수 필드 검증
**경로:** 피드백 작성 모달
1. 받는 사람 필드 비우고 "작성하기" 클릭
2. **기대 결과:** "필수 항목을 모두 입력해주세요" 경고

## 6. 사이클 (Cycles)

### 시나리오 6.1: 새 사이클 생성 ✅
**경로:** 사이클 페이지 → "+ 새 사이클"
1. "+ 새 사이클" 버튼 클릭
2. 모달에서:
   - 사이클명: "2024년 Q2"
   - 설명: "2분기 목표"
   - 시작일: 2024-04-01
   - 종료일: 2024-06-30
3. "만들기" 버튼 클릭
4. **기대 결과:**
   - 모달 닫힘
   - 카드로 사이클 표시
   - 상태: "진행중" (활성 사이클)

### 시나리오 6.2: 사이클 편집 ✅
**경로:** 사이클 페이지
1. 사이클 카드에서 "수정" 버튼 클릭
2. 모달에서 사이클명 수정: "2024년 Q2 → 2024 Q2"
3. "수정하기" 버튼 클릭
4. **기대 결과:**
   - 모달 닫힘
   - 카드의 제목 변경됨

### 시나리오 6.3: 사이클 삭제 ✅
**경로:** 사이클 페이지
1. 사이클 카드에서 "삭제" 버튼 클릭
2. 확인 모달: "이 사이클을 삭제하시겠습니까?"
3. "확인" 클릭
4. **기대 결과:**
   - 사이클이 목록에서 제거됨

### 시나리오 6.4: 날짜 검증
**경로:** 사이클 생성 모달
1. 시작일: 2024-06-30
2. 종료일: 2024-04-01 (시작일보다 이른 날짜)
3. "만들기" 클릭
4. **기대 결과:** "종료일이 시작일보다 나중이어야 합니다" 경고

## 7. 조직도 (Organization)

### 시나리오 7.1: 조직 계층 구조 표시
**경로:** 조직도 페이지
1. 페이지 로드
2. **기대 결과:**
   - 최상위 조직 (개발팀) 표시
   - 팀장 정보 표시
   - 구성원 수 표시
   - 하위 조직 확장/축소 버튼(▶/▼) 표시

### 시나리오 7.2: 하위 조직 확장
**경로:** 조직도 페이지
1. 개발팀의 확장 버튼(▶) 클릭
2. **기대 결과:**
   - 버튼이 ▼로 변경
   - 백엔드팀, 프론트엔드팀 표시
   - 들여쓰기로 계층 표현

### 시나리오 7.3: 새 조직 생성 ✅
**경로:** 조직도 페이지 → "+ 새 조직"
1. "+ 새 조직" 버튼 클릭
2. 모달에서:
   - 조직명: "데이터팀"
   - 팀장: "이데이터"
3. "만들기" 클릭
4. **기대 결과:**
   - 모달 닫힘
   - 새 조직이 목록에 추가됨

### 시나리오 7.4: 조직 편집 ✅
**경로:** 조직도 페이지
1. 조직 카드에서 "수정" 버튼 클릭
2. 팀장 이름 변경
3. "수정하기" 클릭
4. **기대 결과:**
   - 모달 닫힘
   - 카드의 팀장 정보 업데이트

### 시나리오 7.5: 조직 삭제 ✅
**경로:** 조직도 페이지
1. 조직 카드에서 "삭제" 버튼 클릭
2. 확인: "이 조직을 삭제하시겠습니까?"
3. 확인 클릭
4. **기대 결과:** 조직 목록에서 제거됨

## 8. 구성원 관리 (Admin Members)

### 시나리오 8.1: 구성원 목록 조회 ✅
**경로:** 구성원 관리 페이지
1. 페이지 로드
2. **기대 결과:**
   - 테이블 형식으로 구성원 목록 표시
   - 컬럼: 이름, 이메일, 역할, 부서, 작업
   - 역할별 색상 코딩:
     - 관리자: 빨강
     - 팀장: 파랑
     - 구성원: 회색

### 시나리오 8.2: 새 구성원 추가 ✅
**경로:** 구성원 관리 → "+ 구성원 추가"
1. "+ 구성원 추가" 버튼 클릭
2. 모달에서:
   - 이름: "박개발"
   - 이메일: "park@example.com"
   - 역할: "구성원" 선택
   - 부서: "백엔드팀" 입력
3. "추가하기" 클릭
4. **기대 결과:**
   - 모달 닫힘
   - 테이블에 새 구성원 추가됨

### 시나리오 8.3: 구성원 역할 변경 ✅
**경로:** 구성원 관리 → 수정
1. 구성원 행의 "수정" 버튼 클릭
2. 역할 드롭다운에서 "팀장" 선택
3. "수정하기" 클릭
4. **기대 결과:**
   - 테이블에서 역할 업데이트
   - 색상 변경됨 (회색 → 파랑)

### 시나리오 8.4: 구성원 삭제 ✅
**경로:** 구성원 관리
1. 구성원 행의 "삭제" 버튼 클릭
2. 확인: "이 구성원을 삭제하시겠습니까?"
3. 확인 클릭
4. **기대 결과:** 테이블에서 구성원 제거됨

### 시나리오 8.5: 필수 필드 검증
**경로:** 구성원 추가 모달
1. 이름만 입력하고 "추가하기" 클릭
2. **기대 결과:** "필수 항목을 모두 입력해주세요" 경고

## 9. 대시보드 (Dashboard)

### 시나리오 9.1: 대시보드 통계 표시
**경로:** 대시보드 페이지
1. 페이지 로드
2. **기대 결과:**
   - 총 목표 수 표시 (정수)
   - 순항 목표 수 (녹색)
   - 난항 목표 수 (빨강)
   - 완료 목표 수 (파랑)
   - 대기 목표 수 (주황)

### 시나리오 9.2: 진행률 시각화
**경로:** 대시보드 페이지
1. 진행률 섹션 확인
2. **기대 결과:**
   - 순항 퍼센트 표시 (프로그레스 바)
   - 난항 퍼센트 표시 (프로그레스 바)
   - 완료 퍼센트 표시 (프로그레스 바)
   - 각 바의 색상 일치 (녹색, 빨강, 파랑)

### 시나리오 9.3: 최근 활동 표시
**경로:** 대시보드 페이지
1. "최근 활동" 섹션 확인
2. **기대 결과:**
   - 활동 타입별 아이콘 표시 (목표 변경, 체크인, 피드백)
   - 활동 제목, 설명, 타임스탬프 표시
   - 타입별 색상 코딩 (좌측 보더)

### 시나리오 9.4: 빈 대시보드
**전제 조건:** 목표가 없는 상태
1. 목표가 없을 때 대시보드 접속
2. **기대 결과:**
   - 모든 통계가 0으로 표시
   - "최근 활동 없음" 메시지

## 13. 통합 시나리오 (End-to-End)

### 시나리오 13.1: 완전한 OKR 사이클
**시간:** ~10분

1. **로그인**
   - 대시보드 접속

2. **목표 설정**
   - "+ 새 목표" → "마이크로서비스 아키텍처 전환" 생성
   - 가중치 설정: 40%
   - 추가 목표 2개 생성 (가중치 30%, 30%)

3. **체크인**
   - 첫 번째 목표 선택
   - "체크인하기" → 상태 "순항" 선택, 코멘트 입력
   - 제출

4. **피드백 작성**
   - 피드백 페이지로 이동
   - "+ 피드백 작성"
   - 팀원에게 "좋은 점" 피드백 작성

5. **대시보드 확인**
   - 대시보드로 이동
   - 업데이트된 통계 확인

6. **프로필 수정**
   - 프로필 페이지로 이동
   - 부서 정보 수정
   - 저장

**기대 결과:**
- 모든 작업이 정상 완료
- 각 화면에서 최신 데이터 반영됨
